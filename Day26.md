I have successfully designed and built a gamified, single-file, interactive Prior Authorization (PA) Workflow Simulator using vanilla JavaScript, HTML5, and CSS

The goal of this project was to visually map the complex, multi-party logic of the US healthcare prior authorization system into a clean, drag-and-drop state machine
spanning three key operational lanes: Patient, Provider, and Payer.

Building this simulator highlighted a core operational reality: Prior Authorization is less about medical disagreement and more about administrative friction acting as a deliberate architectural constraint.

When translating the data loops into code, the stark misalignment of system incentives became glaringly obvious:

The Provider Ecosystem: Optimized for immediate, comprehensive clinical care delivery.

The Payer Ecosystem: Optimized for cost containment, strict step-therapy validation, and adherence to evidence-based policy guidelines.
